# Чеклист DFIR: Компрометация учётной записи AD и WMI Lateral Movement

## 1. Обнаружение и первичный триаж

- [ ] Проверить алерт в SIEM:
  - [ ] множественные неудачные входы (Event ID 4625) по одной учётной записи;
  - [ ] последующий успешный вход (Event ID 4624) с редкой рабочей станции;
  - [ ] запуск обфусцированного PowerShell на контроллере домена.
- [ ] Зафиксировать:
  - [ ] время событий;
  - [ ] имена хостов;
  - [ ] учётные записи;
  - [ ] связанные VPN/RDP‑подключения.

## 2. Сбор артефактов

- [ ] Образ диска:
  - [ ] создать E01‑образ с помощью FTK Imager;
  - [ ] вычислить SHA‑256;
  - [ ] записать хеш и детали в журнал chain‑of‑custody.

- [ ] Дамп памяти:
  - [ ] снять дамп RAM (WinPMEM / DumpIt);
  - [ ] задокументировать инструмент, версию и параметры.

- [ ] Журналы Windows:
  - [ ] собрать Security, System, Sysmon через KAPE / Velociraptor.

- [ ] Сетевые артефакты:
  - [ ] сохранить PCAP или выгрузить сетевые данные из NTA/EDR;
  - [ ] запустить Volatility `netscan` по дампу памяти.

- [ ] Реестр:
  - [ ] экспортировать релевантные разделы;
  - [ ] прогнать RegRipper по:
    - [ ] `Run`;
    - [ ] `UserAssist`;
    - [ ] `IFEO`.

## 3. Построение временной шкалы (timeline)

- [ ] Определить:
  - [ ] времена VPN/RDP‑подключений (шлюз + контроллер домена);
  - [ ] первый успешный вход (4624);
  - [ ] события запуска PowerShell;
  - [ ] WMI‑активность.
- [ ] Использовать:
  - [ ] Volatility `pslist` / `pstree`;
  - [ ] Volatility `netscan`;
  - [ ] логи Sysmon (процессы, сеть).
- [ ] Учесть различия времени:
  - [ ] задокументировать NTP‑источник;
  - [ ] скорректировать расхождения между хостами и коллекторами.

## 4. Анализ первопричины и масштаба

- [ ] Подтвердить, что имела место фишинговая или иная начальная цепочка компрометации.
- [ ] Проверить статус MFA для затронутого хоста/учётной записи.
- [ ] Идентифицировать механизмы закрепления:
  - [ ] задачи планировщика;
  - [ ] ключи автозапуска в реестре;
  - [ ] сервисы или автозагрузка.
- [ ] Идентифицировать lateral movement через WMI:
  - [ ] с каких хостов;
  - [ ] на какие цели (DC, серверы).

- [ ] Проверка распространения IOC:
  - [ ] повторное использование тех же хешей;
  - [ ] совпадения по YARA;
  - [ ] те же IP/домены/порты на других хостах.
- [ ] Корреляция по всем DC:
  - [ ] просмотреть события входа и сессии на всех контроллерах домена.

## 5. Containment и Eradication

- [ ] Заблокировать/отключить скомпрометированные учётные записи.
- [ ] Инициировать принудительную смену паролей:
  - [ ] для скомпрометированных пользователей;
  - [ ] для связанных админских учёток при необходимости.
- [ ] Отозвать токены и активные сессии.
- [ ] Удалить:
  - [ ] вредоносные задачи планировщика;
  - [ ] ключи реестра, используемые для закрепления;
  - [ ] загруженные бинарники/скрипты.
- [ ] Переместить затронутые хосты из продуктивного OU в карантинный/отдельный OU с усиленным мониторингом.

## 6. Валидация и возврат в продуктив

- [ ] Запустить поиск IOC по:
  - [ ] SIEM;
  - [ ] EDR;
  - [ ] NTA.
- [ ] Проверить:
  - [ ] отсутствие новых аномальных событий входа;
  - [ ] отсутствие подозрительных процессов;
  - [ ] отсутствие подозрительных сетевых соединений.
- [ ] В течение оговорённого периода наблюдения отслеживать состояние.
- [ ] Задокументировать:
  - [ ] все выполненные действия;
  - [ ] финальный статус хостов и учётных записей;
  - [ ] остаточные риски и задачи на доработку.
