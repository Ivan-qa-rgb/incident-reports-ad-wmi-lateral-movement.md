# DFIR чек-лист (Blue Team): компрометация AD-учётки и WMI lateral movement

Сценарий: компрометация доменной учётной записи и дальнейший lateral movement через WMI/PowerShell на контроллер домена и другие критичные серверы.

---

## 1. Обнаружение и первичный триаж

- [ ] Проверить алерт в SIEM:
  - [ ] множественные неудачные входы (Event ID 4625) по одной учётной записи;
  - [ ] последующий успешный вход (Event ID 4624) с редкой/нетипичной рабочей станции;
  - [ ] запуск обфусцированного PowerShell на контроллере домена;
  - [ ] признаки активности WMI/Remote PowerShell.

- [ ] Зафиксировать:
  - [ ] время ключевых событий;
  - [ ] имена хостов (источник, цель, контроллеры домена);
  - [ ] задействованные учётные записи;
  - [ ] связанные VPN/RDP‑подключения (IP, география, устройства).

---

## 2. Сбор артефактов

- [ ] Образ диска:
  - [ ] создать E01‑образ с помощью FTK Imager (или аналога);
  - [ ] вычислить SHA‑256;
  - [ ] записать хеш и детали в журнал chain‑of‑custody (кто/что/когда/куда копировал).

- [ ] Дамп памяти:
  - [ ] снять дамп RAM (WinPMEM / DumpIt);
  - [ ] задокументировать инструмент, версию, параметры, хост и время.

- [ ] Журналы Windows:
  - [ ] собрать Security, System, Sysmon через KAPE / Velociraptor / свои скрипты;
  - [ ] сохранить отдельную копию логов (не полагаться только на то, что уже в SIEM).

- [ ] Сетевые артефакты:
  - [ ] сохранить PCAP или выгрузить сетевые данные из NTA/EDR;
  - [ ] запустить Volatility `netscan` по дампу памяти.

- [ ] Реестр:
  - [ ] экспортировать релевантные разделы (Run/RunOnce, Services, WMI и т.п.);
  - [ ] прогнать RegRipper, минимум по:
    - [ ] `Run`;
    - [ ] `UserAssist`;
    - [ ] `IFEO`.

---

## 3. Построение временной шкалы (timeline)

- [ ] Определить:
  - [ ] времена VPN/RDP‑подключений (шлюз + контроллер домена/целевые хосты);
  - [ ] первый успешный вход (4624) после серии 4625;
  - [ ] события запуска PowerShell;
  - [ ] WMI‑активность (процессы, события, логи).

- [ ] Использовать:
  - [ ] Volatility `pslist` / `pstree`;
  - [ ] Volatility `netscan`;
  - [ ] логи Sysmon (процессы, сеть).

- [ ] Учесть различия времени:
  - [ ] задокументировать NTP‑источник для каждого хоста/коллектора;
  - [ ] скорректировать расхождения между хостами и коллекторами (по возможности привести к единой временной шкале).

---

## 4. Анализ первопричины и масштаба

- [ ] Подтвердить, что имела место фишинговая или иная начальная цепочка компрометации (initial access).
- [ ] Проверить статус MFA для затронутого хоста/учётной записи.

- [ ] Идентифицировать механизмы закрепления (persistence):
  - [ ] задачи планировщика (Task Scheduler);
  - [ ] ключи автозапуска в реестре (Run/RunOnce, Services, IFEO);
  - [ ] сервисы или автозагрузка;
  - [ ] WMI‑подписки (при необходимости).

- [ ] Идентифицировать lateral movement через WMI:
  - [ ] с каких хостов выполнялись WMI‑запросы;
  - [ ] на какие цели (DC, серверы приложений, файловые, SQL и т.п.);
  - [ ] какие команды/скрипты выполнялись (PowerShell, wmic, WMI‑скрипты).

- [ ] Проверка распространения IOC:
  - [ ] повторное использование тех же хешей файлов/процессов;
  - [ ] совпадения по YARA;
  - [ ] те же IP/домены/порты на других хостах.

- [ ] Корреляция по всем DC:
  - [ ] просмотреть события входа и сессии на всех контроллерах домена;
  - [ ] выявить повторяющиеся паттерны использования учёток и источников.

---

## 5. Containment и Eradication

- [ ] Учётные записи:
  - [ ] заблокировать/отключить скомпрометированные учётные записи;
  - [ ] инициировать принудительную смену паролей:
    - [ ] для скомпрометированных пользователей;
    - [ ] для связанных админских/сервисных учёток при необходимости;
  - [ ] отозвать токены и активные сессии (VPN, RDP, Kerberos и т.п.).

- [ ] Хосты:
  - [ ] переместить затронутые хосты из продуктивного OU в карантинный/отдельный OU с усиленным мониторингом;
  - [ ] ограничить сетевой доступ до минимально необходимого для расследования;
  - [ ] при глубокой компрометации планировать переустановку с чистого baseline.

- [ ] Удалить:
  - [ ] вредоносные задачи планировщика;
  - [ ] ключи реестра, используемые для закрепления;
  - [ ] загруженные вредоносные бинарники/скрипты;
  - [ ] по результатам — обновить сигнатуры/правила в EDR/AV/NGFW/IPS под выявленные техники и IOC.

---

## 6. Валидация и возврат в продуктив

- [ ] Запустить поиск IOC по:
  - [ ] SIEM;
  - [ ] EDR;
  - [ ] NTA.

- [ ] Проверить:
  - [ ] отсутствие новых аномальных событий входа;
  - [ ] отсутствие подозрительных процессов (PowerShell, WMI, неизвестные EXE);
  - [ ] отсутствие подозрительных сетевых соединений.

- [ ] В течение оговорённого периода наблюдения:
  - [ ] отслеживать состояние затронутых хостов и учётных записей;
  - [ ] поддерживать повышенный уровень мониторинга (например, 7–14 дней).

- [ ] Задокументировать:
  - [ ] все выполненные действия (кто, что, когда делал);
  - [ ] финальный статус хостов и учётных записей;
  - [ ] остаточные риски и задачи на доработку (детекты, GPO, сегментация, MFA).
